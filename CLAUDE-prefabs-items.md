# CLAUDE-prefabs-items.md — Item & Weapon Prefab Reference

Generated by parsing actual `.prefab` YAML (Session 8). Most prefabs live in **`Assets/GameObject/`** (flat folder, 247 prefabs). Logic item prefabs live in **`Assets/Resources/Defined Items/`** (51).

---

## The four-prefab pattern per item

Every inventory item has up to 4 prefabs:

| Prefab | Folder | Purpose |
|---|---|---|
| **Defined Item** (e.g. `Crossvec`) | `Resources/Defined Items/` | The `ItemBase`/`Firearm` logic prefab instantiated into the player's inventory. Loaded by `InventoryItemLoader` via `Resources.LoadAll<ItemBase>("Defined Items")`. For firearms it carries the **server-side Animator** + `*AnimatorEvents` + attachment components |
| **Viewmodel** (e.g. `CrossvecViewmodel`) | `Assets/GameObject/` | First-person model; instantiated as child of the item instance, rendered by `ViewmodelCamera` |
| **Thirdperson** (e.g. `CrossvecThirdperson`) | `Assets/GameObject/` | What other players see in your hands |
| **Pickup** (e.g. `CrossvecPickup`) | `Assets/GameObject/` | World pickup (NetworkIdentity, spawned by server) |

Pickup and Thirdperson of a weapon share the same `FirearmWorldmodel` content (identical GO counts confirm this — e.g. both Crossvec variants are 112 GOs).

## Firearm viewmodels (`Assets/GameObject/`)

All carry an Animator on a child ("Anims") + `*AnimatorEvents` (client-side `IsServerController=false`). Common extras: `ShootingParticleEffect` (muzzle flash — one of the 2 `OnShotCalled` subscribers), `ReflexSight`, `AnimatedLaser`, `FirearmFlashlight`, `TextAmmoCounter`.

| Viewmodel | GOs | Main script | Notable extras |
|---|---|---|---|
| `AkViewmodel` | 109 | `ClosedBoltFirearmViewmodel` | `CameraRenderFlag`, **Camera×2**, `VolumetricLightRenderer`, SkinnedMesh×20 |
| `Com15Viewmodel` | 79 | `ClosedBoltFirearmViewmodel` | minimal — no extra cameras |
| `Com18Viewmodel` | 103 | `ClosedBoltFirearmViewmodel` | SkinnedMesh×20 |
| `Com45Viewmodel` | 113 | **`Com45Viewmodel`** | |
| `CrossvecViewmodel` | 115 | `ClosedBoltFirearmViewmodel` | `CameraRenderFlag`, **Camera×4**, `ScopeShaderReplacement` |
| `E11SRViewmodel` | 124 | `ClosedBoltFirearmViewmodel` | **Camera×6**, `ScopeShaderReplacement`, `VolumetricLightRenderer`, SkinnedMesh×21 |
| `Fsp9Viewmodel` | 106 | `ClosedBoltFirearmViewmodel` | SkinnedMesh×14 |
| `LogicerViewmodel` | 196 | `ClosedBoltFirearmViewmodel` | **`AnimatedBelt` + `GlowingBarrel`** (the other OnShotCalled subscribers), Camera×4, SkinnedMesh×40 |
| `ShotgunViewmodel` | 82 | **`ShotgunViewmodel`** + `ShotgunAnimatorEvents` | SkinnedMesh×7 |
| `RevolverViewmodel` | 142 | **`RevolverViewmodel`** + `RevolverAnimatorEvents` | Camera×2, SkinnedMesh×14 |
| `DisruptorViewmodel` | 75 | **`DisruptorViewmodel`** + `DisruptorAnimatorEvents` | **Animator×2**, Camera×4, `ScopeShaderReplacement` |

⚠ Scope-equipped viewmodels contain their own Cameras (`CameraRenderFlag`, `ScopeShaderReplacement`) for scope render-to-texture. These are separate from the scene `ViewmodelCamera`.
